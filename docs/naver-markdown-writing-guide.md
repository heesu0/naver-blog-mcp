# 네이버 블로그 HTML 변환을 위한 Markdown 규칙 가이드

이 문서는 Markdown 파일의 **글 구성 방식**을 제한하기 위한 문서가 아니다. 글의 주제, 흐름, 목차 포함 여부, 섹션 구성은 작성자 마음대로 정한다.

이 문서의 목적은 Markdown을 네이버 블로그에 업로드하기 적합한 HTML로 변환할 때, 변환기가 안정적으로 처리할 수 있는 Markdown 작성 규칙과 주의사항을 정리하는 것이다.

## 목표

Markdown 원문을 다음 흐름으로 처리한다.

```text
Markdown 원문
→ 네이버 블로그용 HTML 변환
→ SmartEditor 본문에 HTML 붙여넣기
→ 네이버 블로그 발행
```

따라서 Markdown은 일반 Markdown 문법을 따르되, 네이버 SmartEditor가 보존하지 않는 HTML 기능이나 네이버에서 깨지기 쉬운 표현은 피하는 것이 좋다.

## 핵심 원칙

- Markdown의 내용 구성은 자유롭게 한다.
- 변환 안정성을 위해 표준 Markdown 문법을 우선한다.
- raw HTML에 의존하지 않는다.
- 내부 anchor 링크, script, style block 등은 사용하지 않는다.
- 태그는 Markdown에 넣지 않는 것을 기본으로 한다.
- 복잡한 레이아웃보다 단순한 블록 구조를 사용한다.

## Frontmatter

Frontmatter는 변환 메타데이터로만 사용한다.

### 지원

```md
---
title: 글 제목
---
```

- `title`은 네이버 글 제목으로 사용할 수 있다.
- 본문 H1과 꼭 같을 필요는 없지만, 보통 같게 두면 관리하기 쉽다.

### 태그

네이버 블로그용 Markdown에는 `tags`를 쓰지 않는 것을 기본으로 한다.

```md
---
title: 글 제목
---
```

현재 MCP 동작은 다음과 같다.

- frontmatter `tags`가 있어도 기본적으로 네이버 태그로 사용하지 않는다.
- 변환 라이브러리가 본문 끝에 붙이는 `#태그` 문단은 제거한다.
- 희수님이 명시적으로 발행 옵션 `tags`를 넘긴 경우에만 네이버 태그 필드에 입력한다.

주의: 네이버 블로그가 글이 없는 상태에서 새 글을 발행하면 `#첫글`을 자동으로 붙일 수 있다. 이는 Markdown이나 MCP에서 넣은 태그가 아니다.

## 제목

Markdown 제목은 HTML 제목 태그로 변환된다.

```md
# H1 제목
## H2 제목
### H3 제목
```

현재 변환 예:

```html
<h1 style="font-size: 2em; font-weight: bold;">H1 제목</h1>
<h2 style="font-size: 1.5em; font-weight: bold;">H2 제목</h2>
```

주의:

- 제목에 자동 `id`는 붙지 않는다.
- 직접 `id`를 붙여도 네이버가 저장 과정에서 제거한다.
- 제목을 내부 링크 대상으로 쓰는 클릭 목차는 지원하지 않는다.

## 문단

일반 문단은 단순한 `<p>`로 변환된다.

```md
일반 문단입니다.
```

변환 예:

```html
<p>일반 문단입니다.</p>
```

일반 문단은 네이버 SmartEditor의 기본 스타일에 가깝게 정규화된다.

## 줄바꿈과 빈 줄

Markdown에서 빈 줄은 블록 구분에 사용한다.

```md
첫 번째 문단입니다.

두 번째 문단입니다.
```

변환기는 빈 줄을 네이버에서 보이는 공백 문단으로 일부 반영할 수 있다.

```html
<p>&nbsp;</p>
```

규칙:

- 문단을 나눌 때는 빈 줄 1개를 사용한다.
- 과도한 빈 줄은 피한다.
- 같은 목록 안에서 상위 항목 사이에 빈 줄은 필수가 아니다.

## 목록

### 1단계 불릿 목록

```md
- 항목 A
- 항목 B
- 항목 C
```

네이버에서는 다음처럼 보이도록 변환된다.

```text
• 항목 A
• 항목 B
• 항목 C
```

### 2단계 불릿 목록

하위 항목은 스페이스 2칸을 들여쓴다.

```md
- 상위 항목 A
  - 하위 항목 A-1
  - 하위 항목 A-2
- 상위 항목 B
  - 하위 항목 B-1
```

네이버 표시 예상:

```text
• 상위 항목 A
      ◦ 하위 항목 A-1
      ◦ 하위 항목 A-2
• 상위 항목 B
      ◦ 하위 항목 B-1
```

검증 결과, 상위 항목 사이에 빈 줄을 넣은 경우와 붙여 쓴 경우의 변환 결과는 동일했다.

```md
- A
  - A-1
- B
  - B-1
```

위 형식 그대로 사용 가능하다.

### 번호 목록

```md
1. 첫 번째
2. 두 번째
3. 세 번째
```

번호 목록은 `1.`, `2.`, `3.` 텍스트가 보이는 문단 형태로 변환된다.

## 읽기용 목차

클릭 이동 목차가 아니라 읽기용 목차라면 일반 2단계 불릿 목록으로 작성한다.

```md
## 목차

- 큰 주제 A
  - 하위 주제 A-1
  - 하위 주제 A-2
- 큰 주제 B
  - 하위 주제 B-1
  - 하위 주제 B-2
```

이 형식은 네이버에서 안정적으로 표시된다.

중요: 이 목차는 클릭 이동 기능이 없다. 네이버가 내부 anchor를 제거하기 때문이다.

## 내부 링크와 anchor

다음 방식은 네이버 블로그에서 안정적으로 동작하지 않는다.

```md
[섹션으로 이동](#section)

## 섹션
```

또는 raw HTML:

```html
<a href="#section">섹션으로 이동</a>
<h2 id="section">섹션</h2>
```

실제 발행 검증 결과:

- `id="section"` 제거됨
- `name="section"` 제거됨
- `href="#section"` 보존 안 됨
- 클릭 이동 실패

따라서 같은 글 내부로 이동하는 클릭 목차/내부 링크는 사용하지 않는다.

외부 링크는 사용할 수 있다.

```md
[네이버](https://www.naver.com)
```

## 강조

```md
**굵게**
*기울임*
`인라인 코드`
```

변환 방식:

- `**굵게**` → `<strong style="font-weight: bold;">...`
- `*기울임*` → `<em style="font-style: italic;">...`
- `` `인라인 코드` `` → `<code>` + inline style

인라인 코드는 네이버 공식 코드 컴포넌트가 아니라 HTML 스타일로 코드처럼 보이게 변환된다.

## 코드블록

````md
```ts
const workflow = ['write markdown', 'convert html', 'publish naver'];
console.log(workflow.join(' -> '));
```
````

코드블록은 네이버 공식 코드블록 컴포넌트를 생성하는 방식이 아니다. HTML `<div>`와 inline style로 회색 코드 박스처럼 보이게 변환한다.

현재 표시 특성:

- 회색 배경
- 테두리
- monospace 계열 글꼴
- 줄 단위 표시

규칙:

- 긴 코드는 너무 많이 넣지 않는다.
- 언어 식별자는 가능하면 붙인다. 예: `ts`, `js`, `python`, `bash`
- 복잡한 syntax highlighting에 의존하지 않는다.

## 인용문

```md
> 인용문입니다.
```

변환 결과는 왼쪽 라인과 배경이 있는 박스 형태다.

```html
<div style="border-left: ...; background-color: ...;">
  <p>인용문입니다.</p>
</div>
```

규칙:

- 짧은 요약, 주의사항, 핵심 문장에 사용한다.
- 너무 긴 인용문은 피한다.

## 표

```md
| 단계 | 목적 | 상태 |
|---|---|---|
| Markdown 작성 | 원문 관리 | 완료 |
| HTML 변환 | 네이버 붙여넣기 최적화 | 완료 |
| 블로그 발행 | 실제 화면 검증 | 진행 |
```

표는 HTML `<table>`로 변환된다.

현재 표시 특성:

- 테두리 있음
- 헤더 배경 있음
- 셀 padding 있음

규칙:

- 열은 3~4개 이하를 권장한다.
- 셀 내용은 짧게 쓴다.
- 모바일에서 너무 넓어질 수 있는 표는 피한다.

## 이미지

```md
![이미지 설명](https://example.com/image.png)
```

이미지는 HTML `<img>`로 변환된다.

```html
<img src="..." alt="..." style="max-width: 100%; height: auto;">
```

주의:

- 외부 이미지 URL이 네이버에서 항상 안정적으로 유지된다고 보장할 수 없다.
- 중요한 이미지는 발행 후 반드시 실제 화면에서 확인한다.
- 이미지 직접 업로드 자동화는 별도 개선 대상이다.

## Raw HTML

Markdown 안에 raw HTML을 직접 넣는 것은 가능하더라도 권장하지 않는다.

비권장 예:

```html
<div id="section" class="custom-layout">
  <script>...</script>
</div>
```

이유:

- 네이버가 `id`, `class`, 일부 속성을 제거하거나 바꿀 수 있다.
- SmartEditor가 구조를 재해석한다.
- 발행 후 원래 의도와 다르게 표시될 수 있다.

허용 가능성이 있는 단순 HTML도 실제 발행 전 검증이 필요하다.

## 지원/주의 문법 요약

| Markdown 기능 | 변환/발행 적합성 | 비고 |
|---|---:|---|
| 제목 `#`, `##`, `###` | 좋음 | `id` 자동 생성 없음 |
| 일반 문단 | 좋음 | 네이버 기본 스타일에 가까움 |
| 불릿 목록 | 좋음 | 2단계까지 검증 |
| 번호 목록 | 좋음 | 순서가 필요한 경우 사용 |
| 굵게/기울임 | 좋음 | inline style 적용 |
| 인라인 코드 | 가능 | HTML 스타일 기반 |
| 코드블록 | 가능 | 네이버 공식 코드블록 아님 |
| 인용문 | 좋음 | 박스 스타일 |
| 표 | 가능 | 모바일 폭 주의 |
| 외부 링크 | 좋음 | 일반 링크 |
| 내부 anchor 링크 | 부적합 | 네이버가 제거/변환 |
| 이미지 | 가능 | 실제 화면 검증 필요 |
| raw HTML | 비권장 | 네이버 정규화 영향 큼 |
| frontmatter tags | 비권장 | 기본 운영에서는 쓰지 않음 |

- [ ] raw HTML에 의존하지 않았는가?
- [ ] 목록 하위 항목은 스페이스 2칸으로 들여썼는가?
- [ ] 표가 모바일에서 너무 넓지 않은가?
- [ ] 코드블록이 너무 길지 않은가?

## 최소 예시

```md
---
title: 네이버 블로그 업로드 테스트
---

# 네이버 블로그 업로드 테스트

이 글은 Markdown을 네이버 블로그용 HTML로 변환하는 테스트입니다.

## 목차

- 변환 흐름
  - Markdown 작성
  - HTML 변환
  - 네이버 발행
- 확인 항목
  - 목록 표시
  - 표 표시
  - 코드블록 표시

## 변환 흐름

Markdown 원문을 작성한 뒤 네이버용 HTML로 변환합니다.

## 확인 항목

- 목록은 불릿으로 표시됩니다.
- 하위 목록은 들여쓰기와 다른 불릿으로 표시됩니다.

> 인용문은 박스 형태로 표시됩니다.

```ts
const message = 'hello naver blog';
console.log(message);
```

| 항목 | 결과 |
|---|---|
| 목록 | 정상 |
| 코드블록 | 정상 |
| 표 | 정상 |
```

## 결론

Markdown 파일은 자유롭게 구성하되, 네이버 블로그 업로드에 적합한 HTML로 변환하려면 다음을 지키는 것이 가장 안정적이다.

- 표준 Markdown 문법 사용
- raw HTML 최소화
- 내부 anchor 링크 사용 금지
- 태그는 기본적으로 작성하지 않음
- 목록은 2단계까지 단순하게 작성
- 표/이미지/코드블록은 발행 후 실제 화면 검증
